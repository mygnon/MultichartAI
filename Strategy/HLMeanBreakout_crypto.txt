{===============================================================================
  HLMeanBreakout_crypto
  -------------------------------------------------------------------------------
  PowerLanguage (EasyLanguage / MultiCharts64) trading strategy for crypto.
  Hourly resolution, Data1 only. Long + Short. HIGH/LOW-MEAN channel breakout —
  the levels are the MEAN of recent Highs / Lows +/- ATR buffer (where the
  extremes CLUSTER, not where they peaked) — entered intrabar via STOP +
  ReentryBars cooldown + ATR chandelier trail.

  WHY A NEW BREAKOUT REFERENCE:
  The OOS-robust breakout engine is the same skeleton everywhere — a rolling/non-lagged
  reference LEVEL + an intrabar STOP fill AT that level + a WIDE ATR(14) chandelier
  trailing exit + a post-exit ReentryBars cooldown. The High/Low field can be
  SUMMARIZED by different statistics — this 38th reference completes that gradient:
    - Donchian     : the MAX of the highs      (the single peak)
    - TrimChannel  : the K-th highest          (outlier-trimmed peak)
    - THIS         : the MEAN of the highs     (where the highs CLUSTER)
      UpMean = Average(High, Length) ; DnMean = Average(Low, Length)
      long STOP at (UpMean + BandMult*ATR)[1] ; short at (DnMean - BandMult*ATR)[1]
  The average high is the price zone the market repeatedly probed and stalled at —
  a statistical resistance SHELF rather than a single spike. It sits well INSIDE
  the Donchian extreme, so BandMult carries the confirmation distance (the
  optimizer decides how far beyond the shelf a real break must travel; ~1-3 ATR
  expected). Unlike a Close-based MA center (Keltner), each side is summarized
  from ITS OWN field: resistance from the Highs, support from the Lows — the band
  width adapts to the average bar range organically. Answers: is resistance the
  peak, the trimmed peak, or the cluster?

  This is NOT mean-reversion: it BUYS the upper break and SHORTS the lower break
  (continuation), with the wide ATR trail letting winners run.

    - Long  : flat & cooldown done -> Buy       next bar at prior UpLvl STOP
    - Short : flat & cooldown done -> SellShort  next bar at prior DnLvl STOP
    - Exit  : ATR(14) chandelier trailing stop (the decisive exit). Flat-only.
  Sizing: _Crypto1MUSD = Round(1,000,000 / Close, 0) ~ $1M notional per trade.

  Inputs (all optimizable; names match the optimizer's ParamAxis exactly):
    Length      - averaging window for the High/Low means, bars
    BandMult    - confirmation distance beyond the mean shelf in ATR(14) multiples
    ATRMult     - trailing-stop distance in ATR(14) multiples (wider = looser)
    ReentryBars - bars to stay flat after an exit before a new entry is allowed

  ⚠️ The mean shelf sits INSIDE the range — BandMult near 0 puts the stop level at
     or below the market (instant fills); grids should emphasize ~0.5-4 ATR.

  Defaults: Length=20, BandMult=1.5, ATRMult=3.0, ReentryBars=0
  Author: generated for Tim / MultichartAI, 2026-07-08
===============================================================================}

inputs:
    Length( 20 ),
    BandMult( 1.5 ),
    ATRMult( 3.0 ),
    ReentryBars( 0 ) ;

variables:
    _Crypto1MUSD( 0 ),
    ATRv( 0 ),
    UpMean( 0 ),
    DnMean( 0 ),
    UpLvl( 0 ),
    DnLvl( 0 ),
    flatBars( 0 ),
    TrailLong( 0 ),
    TrailShort( 0 ) ;

{ ---- position size: ~$1M notional per trade, price-normalized ---- }
_Crypto1MUSD = Round( 1000000 / Close, 0 ) ;
if _Crypto1MUSD < 1 then
    _Crypto1MUSD = 1 ;

ATRv = AvgTrueRange( 14 ) ;

{ ---- mean shelves: where the highs / lows cluster ---- }
UpMean = Average( High, Length ) ;
DnMean = Average( Low, Length ) ;

UpLvl = ( UpMean + BandMult * ATRv )[1] ;   { long breakout level  (beyond the high shelf) }
DnLvl = ( DnMean - BandMult * ATRv )[1] ;   { short breakout level (beyond the low shelf)  }

{ ---- flat-cooldown counter: consecutive bars with no open position ---- }
if MarketPosition = 0 then
    flatBars = flatBars + 1
else
    flatBars = 0 ;

{ ---- entries: HL-mean breakout STOP, only when flat + cooldown done ---- }
if MarketPosition = 0 and flatBars >= ReentryBars then
begin
    Buy ( "HmLE" ) _Crypto1MUSD contract next bar at UpLvl stop ;
    SellShort ( "HmSE" ) _Crypto1MUSD contract next bar at DnLvl stop ;
end ;

{ ---- ATR chandelier trailing stop: long ---- }
if MarketPosition = 1 then
begin
    if BarsSinceEntry = 0 then
        TrailLong = EntryPrice - ATRMult * ATRv
    else
        TrailLong = MaxList( TrailLong, Close - ATRMult * ATRv ) ;
    Sell ( "HmLX" ) next bar at TrailLong stop ;
end ;

{ ---- ATR chandelier trailing stop: short ---- }
if MarketPosition = -1 then
begin
    if BarsSinceEntry = 0 then
        TrailShort = EntryPrice + ATRMult * ATRv
    else
        TrailShort = MinList( TrailShort, Close + ATRMult * ATRv ) ;
    BuyToCover ( "HmSX" ) next bar at TrailShort stop ;
end ;
