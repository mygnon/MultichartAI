{===============================================================================
  SFJ_KeltnerTrend_crypto
  -------------------------------------------------------------------------------
  PowerLanguage (EasyLanguage / MultiCharts64) trading strategy for crypto.
  Long + Short. Keltner Channel volatility breakout + ATR chandelier trailing stop.

  Distinct from SFJ_DonchianATR (which uses the prior N-bar High/Low channel):
  this builds the channel from an EMA basis +/- an ATR multiple (a Keltner band),
  so entries trigger on a VOLATILITY breakout relative to the moving average, not
  a raw price-extreme breakout. The EMA basis gives a built-in trend bias (Close
  above EMA+band = strong uptrend), so no separate trend filter is needed.

  Applies the lessons that fixed Donchian v1->v2 on crypto:
    - STOP-breakout entry at the band edge (fills only on a real continuation,
      not a late market order after the close already poked the band),
    - ATR chandelier trailing-stop exit (trend-following exit that lets winners
      run; crypto research showed ATR trailing exits HELP trend strategies),
    - flat-only entry (no always-in flipping) to avoid whipsaw churn.
  Sizing: _Crypto1MUSD = Round(1,000,000 / Close, 0) ~ $1M notional per trade.

  Inputs (all optimizable; names match the optimizer's ParamAxis exactly):
    EMALen   - EMA basis period AND the AvgTrueRange period for the band/stop
    BandMult - Keltner band half-width in ATR multiples (entry distance from EMA)
    ATRMult  - trailing-stop distance in ATR multiples (wider = looser stop)

  Logic:
    Mid   = XAverage(Close, EMALen)
    UpBand = Mid + BandMult * ATR ;  DnBand = Mid - BandMult * ATR
    Long  entry : Buy       next bar at UpBand STOP   (when flat)
    Short entry : SellShort  next bar at DnBand STOP   (when flat)
    Exit  : ATR chandelier trail (ATRMult * ATR) under/over the running extreme

  Defaults: EMALen=20, BandMult=2.0, ATRMult=3.0
  Author: generated for Tim / MultichartAI, 2026-06-23
===============================================================================}

inputs:
    EMALen( 20 ),
    BandMult( 2.0 ),
    ATRMult( 3.0 ) ;

variables:
    _Crypto1MUSD( 0 ),
    ATRv( 0 ),
    Mid( 0 ),
    UpBand( 0 ),
    DnBand( 0 ),
    TrailLong( 0 ),
    TrailShort( 0 ) ;

{ ---- position size: ~$1M notional per trade, price-normalized ---- }
_Crypto1MUSD = Round( 1000000 / Close, 0 ) ;
if _Crypto1MUSD < 1 then
    _Crypto1MUSD = 1 ;

{ ---- Keltner channel (EMA basis +/- ATR band) ---- }
ATRv  = AvgTrueRange( EMALen ) ;
Mid   = XAverage( Close, EMALen ) ;
UpBand = Mid + BandMult * ATRv ;
DnBand = Mid - BandMult * ATRv ;

{ ---- entries: volatility-breakout STOP orders, only when flat ---- }
if MarketPosition = 0 then
begin
    Buy ( "KelLE" ) _Crypto1MUSD contract next bar at UpBand stop ;
    SellShort ( "KelSE" ) _Crypto1MUSD contract next bar at DnBand stop ;
end ;

{ ---- ATR chandelier trailing stop: long ---- }
if MarketPosition = 1 then
begin
    if BarsSinceEntry = 0 then
        TrailLong = EntryPrice - ATRMult * ATRv
    else
        TrailLong = MaxList( TrailLong, Close - ATRMult * ATRv ) ;
    Sell ( "KelLX" ) next bar at TrailLong stop ;
end ;

{ ---- ATR chandelier trailing stop: short ---- }
if MarketPosition = -1 then
begin
    if BarsSinceEntry = 0 then
        TrailShort = EntryPrice + ATRMult * ATRv
    else
        TrailShort = MinList( TrailShort, Close + ATRMult * ATRv ) ;
    BuyToCover ( "KelSX" ) next bar at TrailShort stop ;
end ;
