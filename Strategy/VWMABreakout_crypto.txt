{===============================================================================
  VWMABreakout_crypto
  -------------------------------------------------------------------------------
  PowerLanguage (EasyLanguage / MultiCharts64) trading strategy for crypto.
  Long + Short. VWMA (volume-weighted MA) +/- ATR-band breakout entered intrabar
  via STOP + ReentryBars cooldown + ATR chandelier trail.

  WHY A NEW BREAKOUT REFERENCE:
  The OOS-robust breakout engine is the same skeleton everywhere — a rolling/non-lagged
  reference LEVEL + an intrabar STOP fill AT that level + a WIDE ATR(14) chandelier
  trailing exit + a post-exit ReentryBars cooldown. What varies is only the REFERENCE
  CENTER:
    - Donchian / CloseChannel      : rolling channel EDGE (Highest/Lowest of H/L or C)
    - BollingerBreakout            : SMA center +/- k*stdev
    - Keltner                      : EMA center +/- k*ATR
    - RegChannelBreakout           : linear-regression center +/- k*stdev
    - PivotBreakout / DayChannel   : session-anchored levels (pivot / N-day extremes)
    - MidChannelBreakout           : range-midpoint (HH+LL)/2 +/- k*ATR
    - HullBreakout / KAMABreakout  : low-lag / adaptive-lag MA center +/- k*ATR
    - VWAPBreakout                 : anchored cumulative volume-weighted center
  Two axes are settled: reference SHAPE and center LAG (low-lag wins IS but overfits
  OOS; laggy/adaptive centers are OOS-robust). The VOLUME-information axis is still
  untested. This probes a 12th reference: a rolling VOLUME-WEIGHTED MA center,
      VwmaC = Summation(Close*Volume, Length) / Summation(Volume, Length)
      UpVw = VWMA + BandMult*ATR ; DnVw = VWMA - BandMult*ATR
  VWMA leans toward prices where the market actually TRADED SIZE — high-volume bars
  pull the center, low-volume drift is discounted — so the band recentres on accepted
  value rather than on time-equal-weighted price. Direct A/B: vs Keltner (EMA+/-ATR)
  isolates the volume weighting; vs Hull/KAMA/MidChannel compares center families on
  the identical ATR-band skeleton.

  This is NOT mean-reversion: it BUYS the upper band break and SHORTS the lower band
  break (continuation), with the wide ATR trail letting winners run.

    - Long  : flat & cooldown done -> Buy       next bar at prior UpVw STOP
    - Short : flat & cooldown done -> SellShort  next bar at prior DnVw STOP
    - Exit  : ATR(14) chandelier trailing stop (the decisive exit). Flat-only.
  Sizing: _Crypto1MUSD = Round(1,000,000 / Close, 0) ~ $1M notional per trade.

  Inputs (all optimizable; names match the optimizer's ParamAxis exactly):
    Length      - VWMA window, bars
    BandMult    - breakout distance from the VWMA center in ATR(14) multiples
    ATRMult     - trailing-stop distance in ATR(14) multiples (wider = looser)
    ReentryBars - bars to stay flat after an exit before a new entry is allowed

  ⚠️ Uses the chart's Volume field (crypto feeds report base-asset volume on hourly
     bars; if a feed reports ZERO volume the guard falls back to the plain SMA so
     the strategy degrades to Keltner-like behavior instead of dividing by zero).

  Defaults: Length=20, BandMult=1.0, ATRMult=3.0, ReentryBars=0
  Author: generated for Tim / MultichartAI, 2026-07-06
===============================================================================}

inputs:
    Length( 20 ),
    BandMult( 1.0 ),
    ATRMult( 3.0 ),
    ReentryBars( 0 ) ;

variables:
    _Crypto1MUSD( 0 ),
    ATRv( 0 ),
    VolSum( 0 ),
    VwmaC( 0 ),
    UpVw( 0 ),
    DnVw( 0 ),
    flatBars( 0 ),
    TrailLong( 0 ),
    TrailShort( 0 ) ;

{ ---- position size: ~$1M notional per trade, price-normalized ---- }
_Crypto1MUSD = Round( 1000000 / Close, 0 ) ;
if _Crypto1MUSD < 1 then
    _Crypto1MUSD = 1 ;

ATRv = AvgTrueRange( 14 ) ;

{ ---- VWMA center: volume-weighted mean of the window (zero-volume -> SMA) ---- }
VolSum = Summation( Volume, Length ) ;
if VolSum > 0 then
    VwmaC = Summation( Close * Volume, Length ) / VolSum
else
    VwmaC = Average( Close, Length ) ;

UpVw = ( VwmaC + BandMult * ATRv )[1] ;   { long breakout level }
DnVw = ( VwmaC - BandMult * ATRv )[1] ;   { short breakout level }

{ ---- flat-cooldown counter: consecutive bars with no open position ---- }
if MarketPosition = 0 then
    flatBars = flatBars + 1
else
    flatBars = 0 ;

{ ---- entries: VWMA ATR-band breakout STOP, only when flat + cooldown done ---- }
if MarketPosition = 0 and flatBars >= ReentryBars then
begin
    Buy ( "VwLE" ) _Crypto1MUSD contract next bar at UpVw stop ;
    SellShort ( "VwSE" ) _Crypto1MUSD contract next bar at DnVw stop ;
end ;

{ ---- ATR chandelier trailing stop: long ---- }
if MarketPosition = 1 then
begin
    if BarsSinceEntry = 0 then
        TrailLong = EntryPrice - ATRMult * ATRv
    else
        TrailLong = MaxList( TrailLong, Close - ATRMult * ATRv ) ;
    Sell ( "VwLX" ) next bar at TrailLong stop ;
end ;

{ ---- ATR chandelier trailing stop: short ---- }
if MarketPosition = -1 then
begin
    if BarsSinceEntry = 0 then
        TrailShort = EntryPrice + ATRMult * ATRv
    else
        TrailShort = MinList( TrailShort, Close + ATRMult * ATRv ) ;
    BuyToCover ( "VwSX" ) next bar at TrailShort stop ;
end ;
